September 15, 2025


Experiment to try to auto-update my website using a bash script to edit my .html file, first by appending in a formatted chunk of html with a random word it curls from the web as well as the date/time the script ran.

the formatted html is as follows

<div class="round3">
<p> \ ${timestamp}</p>
<p>random word: \ ${word}</p>
</div>

And then finally my script will git add/commit/push the new file to my websites repo

It took some fiddling to properly automate this. I originally tried placing the .sh script in /etc/cron.daily dir. But it would try and git commit/push from root, which would give the error of unknown author.

Next I tried adding it directly to /etc/crontab but similarly I was getting permission errors, even when I specified to run the script as my user.name.

Finally, the solution was to place it in "crontab -e" which is my usernames personal crontab file.

It would have been better in cron.daily because if my computer isn't on at the set schedueled time, then the script simply won't run. /etc/cron.daily jobs however will be remembered until the system wakes up. Maybe theres a way to get cron.daily to recognize authors but I couldn't seem to find a way and not a lot of information online about it either
