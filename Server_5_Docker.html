Docker
torrenting in containers


Finally got around to learning how to use docker containers. The super simple gist of it is that they are virtual containers that have everything required to run software inside.

My main reason for using this is that it allows me to run my torrent downloading client as well as a vpn all within a container, seperated from the rest of my server. I can't have my torrenting vpn system-wide on my server because I already use another vpn (tailscale) in order to ssh remotely when I am not home. So having only qbt-nox use this vpn is perfect

Up until now all my downloads have been done on my daily driver laptop, and then I will transfer the files through netcat over to my server. The obvious downsides of this being

1.) limited laptop storage, even though the files are only there temporarily
2.) slowish transfers over netcat, especially with 150+ GB at a time
3.) downloads require use of vpn, which means I cannot ssh to work on the server at the same time


Docker was fairly simple to set up to be honest, originally I had made 2 containers. One holding qbittorrent-nox and the other gluetun for vpn. Each worked well individually but I struggled getting qbt-nox container to route through the gluetun one. So I ended up making them all 1 container. This worked well, but then I had issues with connecting to the webui of qbt-nox. Super simple fix is to map the webui port in the gluetun compose.yaml file (i.e 8080:8080) and after doing that everything works as it should. :)


Really convient now for downloading very large files directly onto the hard drive where they will live permanently. Especially for the slow to download ones without many peers


I believe you can also script it so that when a torrent has finished downloading, you can automatically move it to it's appropriate directory for jellyfin to see it (/movies, /television, /anime, etc. etc.)
