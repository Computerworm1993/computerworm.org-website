Setting this up was fun and it gave me the opprotunity to learn a bit about different formatting, like fat32, exfat, and ext4. both the fats are compatible with windows as well as linux, but ext4 is more specific to linux. Windows can still interact with it by using various tools/programs or "Windows Subsytem for Linux". though. Kinda neat.


Anyways rsync is super interesting actually. I have never done real backups like this before, only dragging and dropping files onto a usb. But what makes rsync so amazing is that it will only backup anything thats new or changed, making it wayyyy quicker and more efficient.

How does it know? Well by default, rsync will take a peek at the existing backup files and then what is going to be sent. If the file has a different "last modified" date or if the file size has changed, then it knows that that specific file has changed and needs to be updated/replaced in the backup. If however both of those match up, then the file has not been changed so theres no need to send it to the usb (since the same thing is there already).

The command I like to use is:
rsync -av --delete --exclude='.*' '/home/worm/' '/run/media/usb_name'

breaking this down -av means "archive mode" and "verbose"
--delete means if the file has been deleted off my main disk, then delete it off the backup as well.
--exclude='.*' tells it to not include any file begining with a . (.cache etc etc, any dotfiles)

Another interesting rsync argument you could add if you wanted is -z, which is for compressing the data. Since I'm not really backing up large files and my usb is as large as my disk anyways I personally didnt bother to compress.


Anyways, I also discovered about the use of aliases with bash shell interpreters. I use ZSH so by opening up my .zshrc file (where I have my fastfetch configs and random art scripts as well) I can add aliases for different commands.

An alias is exactly how it sounds, instead of typing in the entire command for rsync to perform the backup I plan to do once a month, I set it to a much easier to remember alias.


Here are a few I have set up. Easier and quicker changing of directories [not my idea, but it is too good not to share. I love these aliases] and then instead of typing out the long rsync command I want i just need to type out "backup-worm" :) how freaking awesome is that.

This also let me delete a couple of the bash scripts I had like for curling weather and QR code since they are single commands I can just set up an alias here and make them even simpler to call


Since I'm backing up files on my laptop, using this rsync alias suites my needs perfectly.
