ABOUT:

This experiment was to try to auto-update my website using a bash script to edit my .html file

first the shell script attempts to curl a random word from the internet as well as stores the time in a variable.

Next it appends these variables into my .html file with properly formatted html, as well as deletes the last chunk so it's only ever 7 days long.

The formated html is as follows

<div class="round3">
<p> \ ${timestamp}</p>
<p>random word: \ ${word}</p>
</div>

And then finally my script will git add/commit/push the new file to my websites repo

This took some fiddling to properly automate. Trying to run from cron.daily would try running it as root, so I'd get git permission errors. The solution was to make a new anacron dir in my home folder and then set it to run from crontab -e with

"@hourly /usr/sbin/anacron -s -t $HOME/.anacron/etc/anacrontab -S $HOME/.anacron/spool"

breaking this down.
@hourly tells it to check once an hour if the job has been ran
-s tells it to serialize the jobs [wait till one is finished before starting the next]
-t tells it to use a specified crontab [my home one] vs the standard one in /etc/
-S likewise specifies where to store spool directory storing timestamps
